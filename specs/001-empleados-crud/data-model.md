# Data Model: CRUD de Empleados

## Entity: Empleado

### Description
Representa un registro de empleado administrado por el sistema.

### Fields
- `clave`: `VARCHAR(16)`
  - Primary Key
  - Autogenerated by system using PostgreSQL sequence `empleados_clave_seq`
  - External format: `E-` + secuencia con padding de 3 dígitos (ej. `E-001`)
  - Immutable after creation
- `nombre`: `VARCHAR(100)`
  - Required
  - Must be non-blank
  - Max length: 100
- `direccion`: `VARCHAR(100)`
  - Required
  - Must be non-blank
  - Max length: 100
- `telefono`: `VARCHAR(100)`
  - Required
  - Must be non-blank
  - Max length: 100

### Validation Rules
- `nombre`, `direccion`, `telefono`:
  - Reject null
  - Reject blank/whitespace-only
  - Reject values longer than 100 characters
- `clave`:
  - Must not be accepted in create payload
  - Must match pattern `^E-[0-9]{3,}$`
  - Must identify an existing row for update/delete/get-by-id operations

### State Transitions
- **Create**: `Empleado` is created with autogenerated `clave` (`E-001`, `E-002`, ...).
- **Update**: Only `nombre`, `direccion`, `telefono` can change.
- **Delete**: Hard delete removes row permanently.

### Relationships
- No relationships with other entities in this feature scope.

## API Payload Models

### CreateEmpleadoRequest
- `nombre`: string (1..100)
- `direccion`: string (1..100)
- `telefono`: string (1..100)

### UpdateEmpleadoRequest
- `nombre`: string (1..100)
- `direccion`: string (1..100)
- `telefono`: string (1..100)

### EmpleadoResponse
- `clave`: string (pattern `^E-[0-9]{3,}$`)
- `nombre`: string
- `direccion`: string
- `telefono`: string

### ErrorResponse
- `timestamp`: string(date-time)
- `status`: integer
- `error`: string
- `message`: string
- `path`: string
